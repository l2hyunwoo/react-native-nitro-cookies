# react-native-nitro-cookies

## Purpose
High-performance HTTP cookie management library for React Native using Nitro Modules JSI architecture. Provides both synchronous and asynchronous APIs for cookie CRUD operations on iOS (NSHTTPCookieStorage / WKHTTPCookieStore) and Android (CookieManager). Published as `react-native-nitro-cookies` on npm. Designed as a drop-in replacement for `@react-native-cookies/cookies` with 5x+ faster performance.

## For AI Agents

### Working In This Directory
- This is a **yarn workspaces monorepo** with two workspaces: `package` and `example`
- Use `yarn` (v3.6.1 via Corepack) for all package management; the `.yarnrc.yml` configures it
- The library source lives in `package/`; treat `example/` as a consumer/test app
- Nitrogen code generator (`nitrogen`) creates C++/Swift/Kotlin bridge code from the TypeScript interface in `package/src/NitroCookies.nitro.ts`
- **Never edit files under `package/nitrogen/generated/`** -- they are auto-generated by running `yarn nitrogen`
- **Never edit files under `package/lib/`** -- they are build output from `react-native-builder-bob`

### Build & Test Commands
- `yarn test` -- run Jest tests
- `yarn lint` -- ESLint across the monorepo
- `yarn typecheck` -- TypeScript type checking
- `yarn nitrogen` -- regenerate Nitrogen bridge code
- `yarn prepare` -- full build (runs `bob build` which triggers nitrogen + module + typescript targets)
- `yarn clean` -- remove Android build and lib output directories

### Commit Conventions
